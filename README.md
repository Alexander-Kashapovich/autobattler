#Autobattler

Симулятор сражений. Все пешки управляются компьютером.

## Модуль навигации
Модуль навигации пешки можно посмотреть в [nv_comp][/game_actors/components/moving/nv_comp.gd]

Тулзы с разметкой можно посмотреть в сцене TestLevel [/world/levels/test_level.tscn]
	- Есть общий полигон
	- Есть частные сектора с отдельным весом, запекаются в ноде Navigation. Делают новый полигон, вычитают себя из шлобального полигона
	- Есть вычитатели (препятствия)

Плюсы:
	- Широкий функционал разметки

Минусы:
	- Требуется перезапекание при появлении новых сегментов.

## Модуль AI
### Nested Inversed FSM
Вложенные конечные автоматы с управлением изнутри.

Каждый узел, кроме Leaf, является независимой fsm. Состояние само запрашивает переход состояние определенного типа на своём урвоне.

Скрипты можно посмототреть в [fsm][/game_actors/fsm]
Посмотреть структуру машины можно в тулзе [/tools/fsm_tools/fsm_visual.tscn]
Если слетел ресурс - загрузить любой рут и нажать go в тулзе.

Плюсы:
	- Интуитивная и читаемая логика переходов.
	- Простое расширение в ширину (на одном уровне)
	- Простая подмена поведения присвоением классу определнного State.ID
	- Можно использовать копии машин, заменив только спрайты в листовых нодах.

Минусы:
	- Крайне канцерное расширение по вертикали.
	- Вызов виртуалки _update() каждый тик даже в пустых состояниях. Сложность линейная, но с оверхедом виртуалки годо может быть чувтсвительно.
	- Каждая машина имеет анти-ООП Blackboard объект, ответственность которого размыта.

### Текущее поведение
Юнит идёт к точке на карте. 
Когда замечает врага - сближается с врагом и атакует. 
У дальников подменено Chase состояние, они дополнительно проверяют безопасность расстояния.
При уроне ниже порога пешка пытается отступить к базе.

Посмотреть на юнитов можно запустив любую из сцен [/world/levels]. Стейты подписаны снизу.

Каждая пешка имеет спеллы. Спелл имеет собственную ценность и набор селекторов/эвалюаторов для оценки конкретного таргета.

Исполнение спелл-таргетинга находится в [/game_actors/fsm/nfsm_leafs/chase], функция is_have_a_good_spell()
Спелл находится в [/game_actors/skills/spell]
Оценщики/селекторы находятся в [/game_actors/components/spell_targeting]

Плюсы:
	- Максимально масштабируемая система поведения

Минусы текущей реализации:
	- Жесткая связка с FSM
	- Могут быть проблемы с выставлением коэффицентов
	- Сложность O(n * ki * m) -> n:спеллы, ki:оценщики каждого спелла, m:таргеты. 
		Сейчас около 2*2*m, но потенциальный рост страшный.
	- Не хватает эвристики для оценки
